# 知乎技术日报 2026-04-25

## Flutter

### [Flutter 2026 Roadmap 发布，未来计划是什么？](https://zhuanlan.zhihu.com/p/2009657387510952563)
**摘要**：Flutter 官方发布 2026 年路线图，核心目标包括：完成 Android Impeller 渲染引擎迁移并移除旧版 Skia 后端（面向 Android 10+），同时确保对 Android 17 及新 iOS 版本的首日支持。Dart 语言层面计划引入 Primary Constructor（简化类声明）和 Augmentations（简化代码生成）。UI 侧将推动 Material 3 Expressive 和 Liquid Glass 风格作为可选包解耦，为自定义 UI 体系提供更灵活的版本管理方式。

### [Flutter 版本选择指南：3.41 发布，稳定的开年 | 2026年2月](https://zhuanlan.zhihu.com/p/2010381509333361060)
**摘要**：Flutter 3.41 于 2026 年 2 月正式发布，被视为年度"开年稳定版"。此版本标志着多窗口支持正式进入成熟期，历经三年迭代终于合并主线，Canonical（Ubuntu 背后的公司）在该功能的推进中贡献显著。文章还对当前各稳定频道版本作了横向对比，并给出不同业务场景下的版本选型建议，适合正在评估升级节点的团队参考。

### [Flutter 鸿蒙 2026 路线发布，加速同步官方生态，进一步优化体验](https://zhuanlan.zhihu.com/p/2016571565735682780)
**摘要**：Flutter-OH（鸿蒙适配分支）社区发布 2026 年路线图，核心目标是大幅缩短与官方 Flutter 版本的同步周期——从此前超过 7 个月压缩至约 4 个月。路线图还包括进一步优化鸿蒙平台的渲染性能与平台 API 接入体验，并持续完善开发者工具链，以帮助现有 Flutter 项目低成本适配鸿蒙生态。

### [Flutter 发布官方 Skills，Flutter 在 AI 领域再添一助力](https://zhuanlan.zhihu.com/p/2012886441642975382)
**摘要**：Flutter 官方推出 Skills 支持，面向 AI Agent 场景提供针对 Flutter 开发的决策逻辑、详细指令与严格约束集合，弥补了 Flutter 此前在官方 AI 能力上的空白。结合 Gemini CLI 和 Claude Code 等 Agent 工具，开发者可通过自然语言描述快速生成和修改 Flutter 组件，进一步降低跨端 UI 开发门槛。

### [新年 Flutter 3.41 就放大招，历经 3 年，多窗口支持终合并](https://zhuanlan.zhihu.com/p/2005389425048307284)
**摘要**：Flutter 多窗口支持功能历经近三年开发，在 3.41 版本中正式合并主线。该功能允许 Flutter 应用在桌面平台同时管理多个独立窗口，并支持窗口间状态共享与路由协调。文章详细梳理了从 RFC 提案到最终落地的完整历程，并感谢 Canonical 团队持续推动该特性，认为这是 Flutter 在桌面领域的重要里程碑。

## Android

### [Android CLI：谷歌为 Android 开发者专研的 AI Agent，提速三倍](https://zhuanlan.zhihu.com/p/2028433276776248769)
**摘要**：Google 发布专为 Android 开发打造的 AI Agent 工具集，包含重新设计的 Android CLI、Android Skills 模块和 Android Knowledge Base。该工具集支持 Gemini in Android Studio、Gemini CLI、第三方 Agent（如 Claude Code、Codex）等多种接入方式。内部实验数据显示，使用 Android CLI 引导 Agent 完成环境配置和项目创建，可将 LLM Token 消耗降低逾 70%，任务完成速度提升约 3 倍。

### [Android 17 有什么需要适配的？2026 Android 禁止侧载又是什么？](https://zhuanlan.zhihu.com/p/2009937286335308608)
**摘要**：文章梳理了 Android 17 的主要 Breaking Changes 和开发者需提前适配的关键点，涵盖权限收紧、新 API 行为变更等内容。同时解析了 Google 的"开发者认证"政策：2026 年 3 月起所有开发者可申请身份认证，9 月起部分地区设备将要求已安装应用来自已认证开发者，未认证应用将受到安装限制，对独立开发者影响较大。

### [Android 不再阻止开发者和高阶用户安装未验证应用，这一政策调整有哪些意义？](https://www.zhihu.com/question/1972267168323298093)
**摘要**：知乎问答热议 Google 对 Android 侧载政策的最新调整——开发者模式下允许安装未经 Play Store 验证的应用，但针对普通用户的限制仍在收紧。讨论聚焦于该政策对开发者测试流程、企业内部分发及开源应用生态的影响，多位开发者认为这是在"安全性"与"开放性"之间寻求新平衡，长远看有利于保留 Android 生态的灵活性。

### [2026 年了，为什么我建议你放弃"纯原生"开发？](https://zhuanlan.zhihu.com/p/2005597393710753297)
**摘要**：作者从工程效率和市场现实两个维度，论述纯原生 Android 开发在 2026 年已不再是大多数项目的最优解。Flutter 的 Impeller 引擎在渲染性能上已与原生高度接近，React Native 新架构也大幅改善了 JSI 调用性能。文章建议团队根据业务场景评估跨平台框架，并给出了不同规模团队的技术选型参考框架，认为"跨平台优先、原生兜底"将成为主流模式。
