## Flutter

### [Flutter 鸿蒙 2026 路线发布，加速同步官方生态，进一步优化体验](https://zhuanlan.zhihu.com/p/2016571565735682780)
**摘要**：Flutter-OH 社区正式发布 2026 年路线规划，核心目标是显著缩短与 Flutter 官方版本的同步周期——从过去平均超过 7 个月压缩到约 4 个月以内，并按季度跟进官方发布节奏。治理层面，Flutter-OH 将主要由 Flutter SIG 推动维护，技术路线通过 SIG 讨论推进。生态层面，2026 年计划完成至少 200 个高优先级三方库的鸿蒙适配，覆盖网络、数据库、图片处理、音视频、地图等常见领域，标志着 Flutter 鸿蒙生态正从验证阶段向工程化、生产化方向加速演进。

### [Flutter 2026 Roadmap 发布，未来计划是什么？](https://zhuanlan.zhihu.com/p/2009657387510952563)
**摘要**：Flutter 官方发布 2026 年路线图，主要聚焦三大方向：一是将 WebAssembly（Wasm）设为 Flutter Web 的默认编译目标，带来更快的启动速度和更低的 JavaScript 开销，但当前 iOS 浏览器（WebKit）和 Firefox/Safari 桌面端仍存在兼容性问题；二是推进 Impeller 渲染引擎覆盖桌面平台（macOS、Windows、Linux），实现六大平台统一渲染引擎；三是改进桌面多窗口支持，提升桌面端的产品级体验。整体方向是"完成未竟之事"，夯实跨平台工程基础。

### [Flutter Web 正式移除 HTML renderer，只支持 CanvasKit 和 SkWasm](https://zhuanlan.zhihu.com/p/11809045697)
**摘要**：Flutter Web 正式移除 HTML 渲染器，现在仅保留两种渲染方案：默认构建使用 CanvasKit（基于 WebGL），WebAssembly 构建使用 SkWasm。移除原因是 HTML renderer 维护成本高、图形表现力受限，而 WebGL 方案已充分成熟。SkWasm 相比 CanvasKit 拥有更优的应用启动时间和帧率表现；体积方面，CanvasKit 附带 1.5MB wasm 文件，SkWasm 为 1.1MB。这一变化标志着 Flutter Web 正式走向 Canvas 渲染路线统一。

**摘要**：Android 17 引入开发者验证（Developer Verification）框架：2026 年 9 月起，部分地区 Android 设备上的应用须由经过认证的开发者发布才可安装，Google Play 上 98% 的应用已自动注册，ADB 安装和企业托管设备豁免不受影响。此外，Android 17 新增高级保护模式（APM），开启后限制侧载、USB 数据传输，并强制 Google Play Protect 扫描。金标联盟（小米、OPPO、vivo、荣耀）已联合要求开发者在 2026 年 7 月 1 日前完成适配，否则应用面临下架风险。

### [Google 发布 Android CLI：打造面向 Android 工程的 Agent 能力](https://zhuanlan.zhihu.com/p/2029480482035737404)
**摘要**：Google 在 I/O 2026 正式发布 Android CLI 1.0，这是一套让 AI 编码 Agent（包括 Claude Code、OpenAI Codex 等）直接调用 Android Studio 工具链的命令行工具集，核心指令覆盖 SDK 安装、工程创建、模拟器管理、应用部署等。配套推出 Android Skills（markdown 格式模块化指令集）和实时知识库（提供最新 Android/Firebase/Kotlin 文档）。官方数据显示，使用后 token 消耗降低超 70%，开发速度提升约 3 倍，标志着 Android 工程开发正式进入 Agentic 时代。

### [Android Gradle Plugin 9.0 发布，为什么这会是个史诗级大坑版本](https://zhuanlan.zhihu.com/p/1997655149036995133)
**摘要**：AGP 9.0 带来多项破坏性变更：完全移除旧 DSL 接口（BaseExtension 等），旧 Variant API（applicationVariants 等）被切断，仅保留新 androidComponents API；运行时要求 Java 17 和 Gradle 9.1.0 及以上版本；KMP 插件不能再与 com.android.application/library 共存于同一模块。这些变化对 Kotlin Multiplatform 用户冲击最大，大量第三方 Gradle 插件和自定义构建脚本将批量失效，需逐一迁移，社区因此称其为"史诗级大坑版本"，强烈建议在升级前仔细评估依赖树。
