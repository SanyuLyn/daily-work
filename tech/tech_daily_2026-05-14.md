## Flutter

### [Flutter 2026 Roadmap 发布，未来计划是什么？](https://zhuanlan.zhihu.com/p/2009657387510952563)
**摘要**：Flutter 官方正式发布 2026 年度路线图，核心目标包括：完成 Impeller 渲染器在 Android 平台的全面迁移，并移除 Android 10 以上设备上的旧版 Skia 后端；保证对 Android 17 及新版 iOS 的第一时间适配支持；持续改善 Web 端和多窗口桌面环境的无障碍体验。Dart 语言层面计划引入 Primary Constructor 简化类声明，以及 Augmentations 机制降低代码生成复杂度。2026 年版本节奏为每季度一个稳定版本，分别在 2 月、5 月、8 月、11 月发布（3.41、3.44、3.47、3.50）。

### [Flutter 3.44 发布前夕，官方宣布 SwiftPM 将完全取代 CocoaPods](https://zhuanlan.zhihu.com/p/2034941553332970100)
**摘要**：在 Flutter 3.44 正式发布前夕，官方宣布将全面转向 Swift Package Manager（SwiftPM），逐步废弃已维护多年的 CocoaPods 依赖管理方案。此举意味着 iOS/macOS 端插件生态将迎来一次依赖管理体系的重构，开发者需要迁移现有基于 CocoaPods 的插件配置。SwiftPM 具备更好的与 Xcode 原生集成、更快的依赖解析速度以及苹果官方的长期支持保障。官方将提供详细的迁移指南，并在过渡期内保持两套方案并存，以减少生态冲击。

### [Flutter 鸿蒙 2026 路线发布，加速同步官方生态](https://zhuanlan.zhihu.com/p/2016571565735682780)
**摘要**：Flutter 鸿蒙（Flutter-OH）社区正式发布 2026 年路线图，核心目标是大幅缩短与 Flutter 官方版本的同步周期——从此前的 7 个月以上压缩至约 4 个月以内。路线图还涵盖对鸿蒙平台三方库的系统性适配工作，包括传感器、亮度控制等常用插件的鸿蒙化改造。与此同时，社区将进一步优化 Flutter 在鸿蒙上的渲染性能，并加强与华为 DevEco Studio 的工具链集成，为国内开发者提供更完整的鸿蒙 Flutter 开发体验。

### [Flutter Web 正式移除 HTML renderer，只支持 CanvasKit 和 SkWasm](https://zhuanlan.zhihu.com/p/11809045697)
**摘要**：Flutter 官方宣布正式从 Flutter Web 中移除 HTML renderer，未来仅保留 CanvasKit 和 SkWasm 两种渲染模式。HTML renderer 虽然资源占用低，但受限于浏览器 DOM 架构，在像素级一致性和复杂 UI 渲染上问题频出。CanvasKit 基于 WebAssembly 与 Skia，SkWasm 则是面向现代浏览器的更轻量化方案，两者均能提供与原生端一致的渲染效果。此变更将影响部分依赖 HTML renderer 低包体积特性的 Web 项目，开发者需尽快评估迁移成本并切换至新渲染管线。

### [2026年Web/App跨端开发方案终极对比：React Native vs Flutter vs Uni-app vs Taro](https://zhuanlan.zhihu.com/p/2010120152784585953)
**摘要**：本文基于 9 年实战经验，对主流跨端开发框架进行系统性横评。结论认为：若追求极致性能与精美 UI，或需要同时覆盖移动端与桌面端，Flutter 是当前最优选择；React Native 在 Web 前端团队复用上有优势，但渲染一致性较弱；Uni-app 适合国内小程序生态优先的场景；Taro 则聚焦 React 技术栈的多端输出。2026 年各框架分化明显，Flutter 凭借 Impeller 渲染器全面落地进一步拉大与竞品的性能差距，成为中大型应用跨端首选。
