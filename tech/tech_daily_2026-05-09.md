## Flutter

### [Flutter 2026 Roadmap 发布，未来计划是什么？](https://zhuanlan.zhihu.com/p/2009657387510952563)
**摘要**：官方发布2026年度路线图，核心目标包括：完成 Impeller 渲染器在 Android 上的全面迁移并彻底移除旧版 Skia 后端（Android 10+）、确保对 Android 17 和 iOS 新版本的第一天支持、推出可选的 Material 3 Expressive 与 Liquid Glass 风格包（支持独立版本锁定）、持续完善 Web 无障碍访问及多窗口桌面环境能力，为开发者绘制了清晰的全年演进蓝图。

### [新年 Flutter 3.41 就放大招，历经3年，多窗口支持终合并](https://zhuanlan.zhihu.com/p/2005389425048307284)
**摘要**：Flutter 3.41 于2026年2月正式发布，多窗口支持功能历经三年开发终于合并入主线，主要由 Canonical（Ubuntu 背后的公司）贡献。新功能涵盖窗口间焦点与键盘事件流转修复、无障碍支持以及每窗口独立生命周期管理，标志着 Flutter 在桌面端能力上迈出重要一步。

### [Flutter 鸿蒙 2026 路线发布，加速同步官方生态，进一步优化体验](https://zhuanlan.zhihu.com/p/2016571565735682780)
**摘要**：Flutter-OH（鸿蒙适配版）发布2026年路线图，承诺跟随季度节奏与官方 Flutter 版本同步，将此前超过7个月的适配周期大幅压缩至约4个月以内。路线图还提出进一步优化渲染性能和平台 API 适配深度，加速与鸿蒙原生生态的融合，为国内鸿蒙用户提供更流畅的跨平台开发体验。

### [Flutter 版本选择指南：3.41 发布，稳定的开年 | 2026年2月](https://zhuanlan.zhihu.com/p/2010381509333361060)
**摘要**：2026年2月版本选择指南系列更新，推荐生产环境使用 3.38.10 稳定版，当前最新版本为 3.41.2。文章详细对比了各版本特性差异与适用场景，指出 3.41 在多窗口、Impeller 稳定性和 Dart FFI 方面有显著提升，为开发者做版本决策提供了清晰参考。

## Android

### [Android 17 有什么需要适配的？2026 Android 禁止侧载又是什么？](https://zhuanlan.zhihu.com/p/2009937286335308608)
**摘要**：Android 17 适配指南详细梳理了开发者必须关注的变更：16K Page Size 强制适配对含 NDK/JNI 原生代码的应用影响最大，需重新编译并充分测试；Google 将于2026年9月起强制要求 Android 设备上的应用来自经过身份验证的开发者，同时"高级侧载"新政将带来六步验证和24小时等待机制，显著提高非官方渠道安装门槛。

### [Google 开始正式强制 Android 适配 16K Page Size，你准备好了吗？](https://zhuanlan.zhihu.com/p/1904196757702834168)
**摘要**：Google 正式宣布强制推进 16K Page Size 适配，主要影响包含原生共享库（.so 文件）的应用。未适配的应用在 16K 内存页设备上将无法正常运行。文章介绍了检测工具使用方法、NDK 编译参数调整方式以及常见适配踩坑点，建议所有含原生代码的应用优先排查并完成迁移。

### [Android Gradle Plugin 9.0 发布，为什么这会是个史诗级大坑版本](https://zhuanlan.zhihu.com/p/1997655149036995133)
**摘要**：AGP 9.0 带来大量破坏性 API 变更，被社区称为"史诗级大坑版本"。核心变化包括：强制开启 R8 全模式（Full Mode）混淆、废弃大量旧版构建 API、调整资源合并与 Manifest 合并逻辑等。对现有大型项目的迁移挑战极大，作者建议提前在独立分支评估影响，制定分阶段升级计划，切勿直接线上迁移。

### [Google 发布 Android CLI：打造面向 Android 工程的 Agent 能力](https://zhuanlan.zhihu.com/p/2029480482035737404)
**摘要**：2026年4月，Google 发布 Android CLI 开发工具套件，由 Android CLI、Android Skills 和 Android Knowledge Base 三部分构成，旨在为 Android 工程引入智能 Agent 能力。开发者可通过自然语言与 Android 开发环境交互，完成代码生成、调试、构建等工作，标志着 AI 辅助开发在官方工具层面的正式落地。

### [iOS/iPadOS 26.3 Beta 2 发布！终结跨系统换机痛点](https://zhuanlan.zhihu.com/p/1994473547167441659)

### [如何评价引入 Liquid Glass 设计的 iOS 26？](https://www.zhihu.com/question/1915597038030128925)
**摘要**：苹果在 iOS 26 中全面引入 Liquid Glass（流体玻璃）设计语言，UI 元素呈现出半透明玻璃质感与流体动态效果。知乎社区评价褒贬不一：设计师群体认可其视觉革新，认为是继扁平化以来最大的设计范式转变；但也有用户担忧半透明背景影响文字可读性，以及高动态效果对视觉敏感人群不友好，引发无障碍访问方面的讨论。

### [苹果即将发布 iOS 26.4.2 新系统更新！](https://zhuanlan.zhihu.com/p/2030073766609417990)
**摘要**：苹果在 iOS 26.4.1 发布后短期内即预告 iOS 26.4.2 即将推出，该版本聚焦安全漏洞修复和系统稳定性改善，包含对工信部此前提示的高危漏洞的补丁。苹果建议所有用户尽快升级以保护设备安全，同时本次更新也修复了部分用户反馈的"连续互通"功能异常问题。
