## Flutter

### [Flutter 3.41.6 版本很重要，你大概率需要更新一下](https://zhuanlan.zhihu.com/p/2021619335085454240)
**摘要**：Flutter 3.41.6 是一个重要的 hotfix 版本，专门修复了一个长期存在的 Android ANR（应用无响应）问题。该 bug 在 Android 16 的 2026 年 3 月安全更新后更加容易复现：当设备锁屏或应用切入后台时（尤其是播放视频期间），Flutter 的 Impeller 渲染引擎（Vulkan 后端）在调用 `vkAcquireNextImageKHR` 失败后，Fence 对象未被正确处理，导致 UI 主线程被永久阻塞。修复方案确保仅在 GPU 帧提交成功后才将 Fence 标记为 pending，若获取失败则跳过等待。强烈建议使用 Impeller + Vulkan 后端的开发者尽快升级至该版本。

### [Flutter 2026 Roadmap 发布，未来计划是什么？](https://zhuanlan.zhihu.com/p/2009657387510952563)
**摘要**：Flutter 团队发布了 2026 年官方 Roadmap，核心目标包括：完成 Android 平台 Impeller 渲染器的全面迁移，并对 Android 10+ 设备移除遗留的 Skia 后端；保证对 Android 17 和即将发布的 iOS 新版本的 Day-one 支持；持续推进 Web 平台性能提升和多窗口桌面端无障碍功能的完善。此外，Material 3 Expressive 和 Liquid Glass 风格将作为可选独立包提供，采用弹性语义化版本管理，为设计系统解耦重构奠定基础。整体路线图体现了 Flutter 在跨平台一致性与平台原生适配之间寻求平衡的战略方向。

### [Flutter 鸿蒙 2026 路线发布，加速同步官方生态，进一步优化体验](https://zhuanlan.zhihu.com/p/2016571565735682780)
**摘要**：Flutter-HarmonyOS（Flutter-OH）团队发布 2026 年适配路线图，核心策略是将官方版本同步周期从过去的超过 7 个月大幅压缩至约 4 个月，实现每季度跟进一次官方 Flutter 稳定版的节奏。路线图还承诺进一步优化鸿蒙特有平台能力的集成体验，减少与官方 Flutter API 的差异，提升开发者在多平台（含鸿蒙）项目中的代码复用率。这一举措对使用 Flutter 开发鸿蒙应用的团队意义重大，意味着可以更快用上 Flutter 新特性，同时降低维护多套分支的成本。

### [新年 Flutter 3.41 就放大招，历经 3 年，多窗口支持终合并](https://zhuanlan.zhihu.com/p/2005389425048307284)
**摘要**：历经 3 年开发，Flutter 3.41 正式合并了桌面端多窗口支持功能（由 Canonical/Ubuntu 团队主导贡献）。新版本在多窗口场景下显著改善了键盘焦点切换逻辑、无障碍辅助功能支持，以及各窗口独立的生命周期管理机制。开发者现在可以为桌面应用创建和管理多个独立窗口，每个窗口拥有独立的 Widget 树和状态。这一里程碑特性让 Flutter 在桌面端的竞争力大幅提升，也为复杂生产力工具类应用的构建提供了原生级别的窗口管理体验。

## Android

### [Google 发布 Android CLI：打造面向 Android 工程的 Agent 能力](https://zhuanlan.zhihu.com/p/2029480482035737404)
**摘要**：2026 年 4 月，Google 发布 Android Agent 开发三件套：Android CLI、Android Skills 和 Android Knowledge Base，旨在统一不同 AI 开发环境（Android Studio、Gemini CLI、Claude Code 等）中的 Android 工程质量。Android CLI 可将 LLM Token 使用量降低 70%，使 Android 开发任务完成速度提升约 3 倍。核心命令包括：`android sdk install`（精准下载所需 SDK 组件）、`android create`（从官方模板快速生成项目）、`android emulator`（管理虚拟设备）。目前处于 Preview 阶段，与 Android Studio 形成互补定位：先用 Agent 快速搭原型，再回 Android Studio 做深度调试和工程化。

### [Android 17 有什么需要适配的？2026 Android 禁止侧载又是什么？](https://zhuanlan.zhihu.com/p/2009937286335308608)
**摘要**：Android 17 引入多项重要变更。开发者身份验证（Developer Verification）框架要求所有上架应用绑定经验证的开发者账号，Google 计划 2026 年 9 月在部分地区强制执行（中国大陆暂不受影响，ADB 安装和企业托管设备豁免）。大屏适配方面，当设备 sw > 600dp（折叠屏展开态、平板等）时，`screenOrientation`、`resizableActivity` 等属性将被系统强制忽略，必须支持自由窗口模式。小米、OPPO、vivo、荣耀等金标联盟成员已呼吁开发者在 **2026 年 7 月 1 日前**完成适配，逾期有下架风险。

### [Android Gradle Plugin 9.0 发布，为什么这会是个史诗级大坑版本](https://zhuanlan.zhihu.com/p/1997655149036995133)
**摘要**：Android Gradle Plugin (AGP) 9.0 发布，被业内称为"史诗级大坑版本"，因其包含大量破坏性变更。AGP 9.0 要求 Gradle 最低版本同步升级，并移除了多个此前标记为 Deprecated 的 API，同时对构建缓存和增量编译机制进行了重构。对于大型多模块项目，升级后可能触发大量编译错误，需要同步升级第三方插件和依赖库。主要坑点涉及 Variant API 变更、资源合并策略调整、BuildConfig 生成逻辑重构等。文章提供了详细的分步迁移建议，适合正在评估是否升级 AGP 的团队提前研读风险。

### [如何看待谷歌逐步收紧安卓应用侧载计划？](https://www.zhihu.com/question/2019018192186557023)
**摘要**：谷歌于 2026 年 3 月正式宣布"Android 高级侧载流程"（Advanced Sideloading Process），为未经开发者验证的应用引入六步确认流程和 24 小时强制等待期。这一举措被认为是谷歌在安全性与开放性之间寻找新平衡点——既回应监管机构对应用生态安全的关切，又试图避免像苹果那样被指责完全封闭。知乎讨论中，开发者和用户意见分歧明显：支持者认为有助于减少恶意软件传播，批评者则担心这会逐步侵蚀 Android 的开放性优势，并对非 Google Play 渠道的应用生态造成打击。
