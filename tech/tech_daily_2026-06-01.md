# 技术日报 2026-06-01

## Flutter

### [Flutter 发布官方 Skills，Flutter 在 AI 领域再添一助力](https://zhuanlan.zhihu.com/p/2012886441642975382)
**摘要**：Flutter 官方发布了 flutter/skills 项目，为 AI Agent 提供标准化的 Flutter 开发操作流程。该项目包含 Flutter MCP（Model Context Protocol）和 GenUI 等工具，通过定义系列规范化步骤，让 AI 能够准确执行 Flutter 开发任务，包括组件生成、代码优化和界面设计等。这是 Flutter 官方在 AI 辅助开发领域的重要布局，标志着 Flutter 生态与 AI 工具链的深度融合正式起步。

### [Flutter 鸿蒙 2026 路线发布，加速同步官方生态，进一步优化体验](https://zhuanlan.zhihu.com/p/2016571565735682780)
**摘要**：Flutter 鸿蒙社区正式发布 2026 年路线图。核心目标是大幅缩短与官方 Flutter 版本的同步差距——从此前超过 7 个月压缩到约 4 个月，并逐步实现季度对齐。路线图还涵盖鸿蒙平台特有的体验优化方向，包括平台 API 覆盖度提升、性能调优，以及对最新 HarmonyOS 系统能力的适配计划，为开发者提供更及时的跨平台支持。

### [Flutter Web 正式移除 HTML renderer，只支持 CanvasKit 和 SkWasm](https://zhuanlan.zhihu.com/p/11809045697)
**摘要**：Flutter 官方宣布正式移除 Flutter Web 的 HTML renderer，未来只支持 CanvasKit 和 SkWasm 两种渲染引擎。此举将统一 Web 端的渲染路径，显著提升渲染一致性与性能上限。SkWasm 基于 WebAssembly 技术，能带来接近原生的运行效率。该变更预计在 beta 版本中于 2026 年底正式合并，开发者需提前排查并迁移依赖 HTML renderer 的相关代码。

### [Flutter 2026 Roadmap 发布，未来计划是什么？](https://zhuanlan.zhihu.com/p/2009657387510952563)
**摘要**：Flutter 2026 官方路线图正式发布，三大核心方向清晰：一是持续完善 Impeller 渲染引擎，提升各平台渲染稳定性；二是计划将 WebAssembly（Wasm）定为 Flutter Web 的默认运行时，以提供近原生的性能体验；三是探索在 Dart 运行时解释字节码的能力，支持动态代码下发，为热更新等场景奠定基础。整体方向聚焦性能、Web 和灵活部署三个维度。

## Android

### [Android 17 有什么需要适配的？2026 Android 禁止侧载又是什么？](https://zhuanlan.zhihu.com/p/2009937286335308608)
**摘要**：文章系统梳理了 Android 17 的关键适配要点，并重点解读了 Google 在 2026 年推出的开发者认证机制——未通过认证的应用将被限制安装（即"禁止侧载"政策）。这一变化对独立开发者和企业内部应用分发影响较大，开发者需了解认证流程和合规要求。文章还涵盖了 targetSdkVersion 升级要求、新权限模型及后台行为限制等核心适配项。

### [Android 17 新适配要求，各大权限进一步收紧，适配难度提升](https://zhuanlan.zhihu.com/p/2020865242960277920)
**摘要**：Android 17 在权限管理上进行了更严格的收紧，多项敏感权限的申请和使用条件均有变化，整体适配难度相比 Android 16 明显提升。文章逐一列举了受影响的权限类别，包括位置、通知、存储等，并分析了各项变化对现有 App 的潜在影响。建议开发者提前进行权限审查，避免在正式版发布后出现功能异常或被下架的风险。

### [Android 宣布 Runtime 编译速度史诗级提升：在编译时间上优化了 18%](https://zhuanlan.zhihu.com/p/1984226146229954374)
**摘要**：Android 官方宣布 Android Runtime（ART）在编译速度上实现重大突破，整体编译时间优化幅度达到 18%，且未对代码质量或峰值内存使用量产生负面影响。这一提升将直接改善 App 安装速度和冷启动体验。技术层面主要得益于编译器中间表示（IR）优化及更高效的并行编译策略，对低端设备的体验改善尤为明显。
