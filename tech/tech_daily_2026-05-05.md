# 技术日报 2026-05-05

## Flutter

### [鸿蒙版 Flutter 三方库适配案例【screen_brightness】](https://zhuanlan.zhihu.com/p/29600343631)
**摘要**：本文以 screen_brightness 插件为例，详细介绍了如何将 Flutter 三方库适配到 HarmonyOS（鸿蒙）平台。文章涵盖了适配的整体思路、平台通道接入方式、以及鸿蒙特有 API 的调用方法，为开发者提供了一套可复用的鸿蒙 Flutter 三方库适配参考模板，对推动 Flutter 生态在鸿蒙平台落地具有实践意义。

### [Flutter-Web 从 0 到部署上线（实践 + 埋坑）](https://zhuanlan.zhihu.com/p/1938276051617845509)
**摘要**：本文系统梳理了 Flutter Web 从项目搭建到上线部署的完整流程，重点分析了 Flutter Web 的适用场景与核心优势（跨平台零额外学习成本、渲染一致性）和劣势（包体积偏大、部分浏览器兼容性问题）。同时分享了作者在实际项目中踩过的坑及解决思路，对计划将 Flutter 应用扩展到 Web 端的开发者有较高参考价值。

### [Flutter Web 正式移除 HTML renderer，只支持 CanvasKit 和 SkWasm](https://zhuanlan.zhihu.com/p/11809045697)
**摘要**：Flutter 官方正式宣布移除 Flutter Web 的 HTML renderer，今后仅支持 CanvasKit 和 SkWasm 两种渲染方案。CanvasKit 基于 WebAssembly 与 Skia，渲染效果更接近原生；SkWasm 是新一代更轻量的 WASM 渲染器，性能更优。此变更对存量 Flutter Web 项目有一定迁移成本，开发者需尽快完成渲染器切换，避免后续版本升级受阻。

### [Flutter 多版本管理工具 Puro，它和 FVM 有什么区别？](https://zhuanlan.zhihu.com/p/49313090195)
**摘要**：本文对比了 Flutter 多版本管理工具 Puro 与 FVM 的区别。Puro 采用共享缓存机制，多个项目可复用同一 Flutter SDK 的公共组件，大幅减少磁盘占用；切换速度也显著快于 FVM。文章还介绍了 Puro 的安装配置方法和常用命令，适合需要在多个 Flutter 版本之间频繁切换的开发者参考。

### [Flutter 官方多窗口体验，为什么 Flutter 推进那么慢，而 CMP 却支持那么快](https://zhuanlan.zhihu.com/p/1912806685069771447)
**摘要**：文章分析了 Flutter 官方多窗口功能进展迟缓的原因：Flutter 的跨平台渲染架构使多窗口实现较为复杂，需协调多个平台层。相比之下，Compose Multiplatform（CMP）依托 Kotlin 原生生态，可更直接地复用平台多窗口能力。作者呼吁 Flutter 团队加快多窗口支持落地，以应对桌面和平板端日益增长的需求。

## Android

### [Android 17 有什么需要适配的？2026 Android 禁止侧载又是什么？](https://zhuanlan.zhihu.com/p/2009937286335308608)
**摘要**：文章整理了 Android 17 的关键适配要点，并解读了 Google 2026 年逐步推行的侧载管控计划。根据 Google 时间表，2026 年 3 月开放开发者身份验证，9 月起部分地区 Android 设备上未经认证开发者发布的应用将无法安装。此外，Android 17 引入了新的权限模型与后台行为限制，开发者需提前评估影响并完成适配。

### [Google 发布 Android CLI：打造面向 Android 工程的 Agent 能力](https://zhuanlan.zhihu.com/p/2029480482035737404)
**摘要**：Google 于 2026 年 4 月正式发布 Android Agent 开发工具包，核心组件包括 Android CLI、Android Skills 和 Android Knowledge Base。该工具包旨在让开发者无论使用 Android Studio、Gemini CLI 还是 Claude Code 等第三方 Agent，都能稳定获得高质量的 Android 工程输出。这标志着 Google 正式将 AI Agent 能力引入 Android 开发工作流。

### [Google 开始正式强制 Android 适配 16K Page Size，你准备好了吗？](https://zhuanlan.zhihu.com/p/1904196757702834168)
**摘要**：Google 正式要求 Android 应用适配 16K 页面大小（Page Size），以支持新一代 ARM 处理器的内存管理优化。16K Page Size 可显著提升系统性能和内存效率，但也对 NDK 原生代码、内存对齐和 SO 库的编译方式提出了更高要求。使用 C/C++ 原生库的应用开发者需重点排查并完成兼容性适配。

### [Android Gradle Plugin 9.0 发布，为什么这会是个史诗级大坑版本](https://zhuanlan.zhihu.com/p/1997655149036995133)
**摘要**：Android Gradle Plugin（AGP）9.0 带来了三项重大变化：全面切换至新 DSL 接口、废弃旧版 Variant API 和 Implementation 接口、将内置 Kotlin 设为默认编译器。这些变化对存量项目的构建脚本影响较大，许多旧有配置写法将不再兼容。作者提醒开发者谨慎升级，并提供了常见报错的排查和迁移建议。
