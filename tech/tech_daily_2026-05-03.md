### [Flutter 版本选择指南：3.35 稳定，3.38 发布 | 2025年11月](https://zhuanlan.zhihu.com/p/1977423333223048923)
**摘要**：文章梳理了当前 Flutter 各版本的现状：3.35 已进入稳定渠道，三方库适配较为完善，适合生产项目；3.38 已正式发布但生态跟进仍需时间。作者建议对版本敏感的团队优先采用 3.35，待主要依赖库完成适配后再升级至 3.38，并列举了升级过程中常见的破坏性变更及应对方案。

### [Flutter 开发在 Cursor & Trae 不一样的 AI 真实体验](https://zhuanlan.zhihu.com/p/26655566907)
**摘要**：作者分享了在实际 Flutter 项目中同时使用 Cursor 和 Trae 两款 AI 编程工具的真实感受，重点对比了两者在将 Redux 状态管理迁移至 Riverpod 场景下的代码生成质量、上下文理解能力及交互流畅度。总体结论是两款工具各有侧重，Cursor 在大范围重构上表现更稳定，而 Trae 对国内开发者的中文需求响应更友好。

### [Flutter Web 正式移除 HTML renderer，只支持 CanvasKit 和 SkWasm](https://zhuanlan.zhihu.com/p/11809045697)
**摘要**：Flutter 团队宣布正式从 Flutter Web 中移除 HTML renderer，后续仅支持 CanvasKit 和 SkWasm 两种渲染后端。CanvasKit 提供接近原生的渲染效果但包体较大，SkWasm 则在现代浏览器中具备更优异的性能表现。文章分析了这一决策对现有 Web 项目的迁移影响，以及如何评估并选择合适的渲染器。

### [Flutter-Web 从 0 到部署上线（实践+埋坑）](https://zhuanlan.zhihu.com/p/1938276051617845509)
**摘要**：作者完整记录了一个 Flutter Web 项目从本地开发到云端部署的全流程，涵盖环境配置、路由适配、跨域处理、CDN 静态资源优化等关键环节，并汇总了实践中踩过的常见坑点，包括 CanvasKit 资源加载慢、SEO 不友好等问题及其解决方案，适合首次上线 Flutter Web 的开发者参考。

## Android

### [Android 17 有什么需要适配的？2026 Android 禁止侧载又是什么？](https://zhuanlan.zhihu.com/p/2009937286335308608)
**摘要**：文章系统梳理了 Android 17 的主要适配要点：`usesCleartextTraffic` 属性被正式废弃，需改用 Network Security Config 白名单 HTTP 域名；同时介绍了 Google 自 2026 年起推行的"安卓高级侧载流程"政策——用户在非官方渠道安装应用需经历六步身份核验并强制等待 24 小时，此举旨在打击恶意应用，但也引发了开发者社区的广泛讨论。

### [Google 发布 Android CLI：打造面向 Android 工程的 Agent 能力](https://zhuanlan.zhihu.com/p/2029480482035737404)
**摘要**：Google 于 2026 年 4 月正式发布 Android CLI、Android Skills 和 Android Knowledge Base 套件，使 AI 助手（包括 Claude Code）能够在各类 IDE 中执行一致的 Android 工程任务，如代码生成、测试运行、构建调试等。这标志着 Android 开发工具链全面向 Agentic 模式演进，开发者可通过自然语言驱动完整的 Android 工程流水线。

### [Android Gradle Plugin 9.0 发布，为什么这会是个史诗级大坑版本](https://zhuanlan.zhihu.com/p/1997655149036995133)
**摘要**：Android Gradle Plugin 9.0 带来了多项破坏性改动，包括移除大量已废弃 API、对变体配置方式的重构以及对部分旧版 DSL 的强制淘汰，导致大量存量项目升级后出现编译失败。文章逐一分析了高频报错原因，提供了迁移 checklist，并建议中大型项目在升级前进行充分的依赖链排查，避免在关键交付节点踩坑。

### [Google 开始正式强制 Android 适配 16K Page Size，你准备好了吗？](https://zhuanlan.zhihu.com/p/1904196757702834168)
**摘要**：Google 正式要求新上架的 Android 应用必须支持 16K 内存页大小（Page Size），以匹配新一代 ARM 芯片的硬件规格并提升内存效率。文章讲解了 16K Page Size 的技术背景、对 NDK 原生库的具体影响，以及如何检测并修复不兼容的 `.so` 文件，是 NDK 开发者的必读适配指南。

### [如何看待谷歌逐步收紧安卓应用侧载计划？](https://www.zhihu.com/question/2019018192186557023)
**摘要**：多位知乎用户从不同角度探讨了 Google 收紧 Android 侧载的政策动向。支持者认为此举能有效降低恶意软件传播风险，尤其保护非技术用户；反对者则担忧开放生态受损，独立开发者和企业内部分发渠道将面临更高合规成本。整体评论偏向认为这是 Google 向 iOS 封闭模式靠拢的信号。
