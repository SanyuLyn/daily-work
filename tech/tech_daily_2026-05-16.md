## Flutter

### [Flutter 鸿蒙 2026 路线发布，加速同步官方生态，进一步优化体验](https://zhuanlan.zhihu.com/p/2016571565735682780)
**摘要**：Flutter-OH 计划在 2026 年将适配周期压缩到约 4 个月以内，相比过去平均超 7 个月有显著改善，并按季度跟进官方版本发布。2026 年社区规划至少 200 个高优先级 Flutter 三方库的鸿蒙适配，覆盖网络、数据库、图片处理、音视频、地图等常见领域。目前稳定版本（3.7.12、3.22.0、3.27.4）与官方 3.41 版本仍存在小版本差距，关键看点在于 Flutter-OH 渲染后端与鸿蒙原生 ArkUI 引擎的深度集成进展。

### [Flutter Web 正式移除 HTML renderer，只支持 CanvasKit 和 SkWasm](https://zhuanlan.zhihu.com/p/11809045697)
**摘要**：Flutter Web 官方宣布正式移除 HTML 渲染器，后续只支持 CanvasKit 和 SkWasm 两种渲染方式。CanvasKit 基于 WebAssembly 和 Skia 图形库，提供接近原生应用的渲染质量；SkWasm 则通过 Web Workers 进一步优化渲染性能。这一变更提升了渲染一致性与性能，但也要求开发者迁移依赖 HTML 渲染器特性的代码，需提前进行兼容性测试。

### [Flutter3.41 发布，鸿蒙如何选择？](https://zhuanlan.zhihu.com/p/2005202488488378695)
**摘要**：Flutter 官方 3.41 版本带来多项性能改进和新特性，文章分析了鸿蒙开发者面临的版本选择困境。Flutter-OH 当前稳定版本滞后于官方版本，开发者需在功能完整性与鸿蒙系统兼容性之间权衡。文章对比了使用官方 Flutter 版本与使用 Flutter-OH 分支的利弊，并针对不同业务场景给出了选择建议。

### [鸿蒙版 Flutter 三方库适配案例【screen_brightness】](https://zhuanlan.zhihu.com/p/29600343631)
**摘要**：文章以 screen_brightness 插件为例，详细介绍了 Flutter 三方库适配鸿蒙平台的完整过程：分析原有插件的原生 API 调用、用鸿蒙 ArkTS 重写平台通道实现、处理 API 差异与兼容性问题。此类社区实战案例为推动 Flutter 鸿蒙生态快速扩张提供了可复用的方法论参考。

### [Flutter-Web 从 0 到部署上线（实践+埋坑）](https://zhuanlan.zhihu.com/p/1938276051617845509)
**摘要**：文章从零开始讲解 Flutter Web 项目的搭建与上线部署，总结了实际开发中遇到的坑。涵盖 Flutter Web 的适用场景分析、性能瓶颈（首屏加载时间长、JavaScript 互操作开销）、SEO 限制等问题，以及完整的 CI/CD 部署方案。对于计划将现有 Flutter App 扩展到 Web 端的开发者具有较高参考价值。

## Android

### [Android 17 有什么需要适配的？2026 Android 禁止侧载又是什么？](https://zhuanlan.zhihu.com/p/2009937286335308608)
**摘要**：文章系统梳理了 Android 17 的核心适配要点，以及 Google 2026 年推行的禁止非官方侧载计划。Android 17 将引入更严格的权限管控，开发者需提前处理存储、通知等敏感权限的变更。Google 计划要求 Android 设备在部分地区实施六步骤验证 + 24 小时等待期才能安装侧载应用，显著提高分发门槛，引发开发者和用户广泛关注。

### [Google 开始正式强制 Android 适配 16K Page Size，你准备好了吗？](https://zhuanlan.zhihu.com/p/1904196757702834168)
**摘要**：Google 宣布将强制要求 Android 应用适配 16KB 内存页大小，主要影响使用 NDK 的原生代码开发者。16K Page Size 可提升系统内存效率和应用运行性能，但部分依赖 4KB 页大小假设的第三方库需要重新编译。文章详细列出了受影响的代码场景，并提供了迁移指南和工具推荐，建议开发者尽早评估影响范围并着手适配。

### [Android "16.5" 自带反广告，但大家恐怕会失望](https://zhuanlan.zhihu.com/p/1981468008783291529)
**摘要**：Android 16 QPR2（即 "Android 16.5"）内置了广告拦截相关功能，引发用户期待。然而文章指出该功能实际效果十分有限，并非全局广告屏蔽，而是局限于特定场景且需要开发者主动适配才能生效。与第三方广告拦截方案相比，系统级实现有其优势，但当前落地效果令人失望，用户短期内无法获得理想的无广告体验。

### [Google 推出耗电应用标记｜Android 出海政策周报](https://zhuanlan.zhihu.com/p/1982391265216701342)
**摘要**：Google Play 将于 2026 年开始为高耗电应用添加显眼标签，帮助用户识别对设备续航影响较大的应用。同期新政还要求开发者完成身份认证——2026 年 9 月起，部分地区的应用需要开发者通过身份验证才能上架，对出海开发者影响较大。本期周报还梳理了 Google Play 近期政策调整对海外应用分发生态的整体影响。

### [如何看待谷歌逐步收紧安卓应用侧载计划？](https://www.zhihu.com/question/2019018192186557023)
**摘要**：知乎讨论汇聚了开发者和用户对谷歌收紧 Android 侧载政策的多方观点。支持者认为此举有助于减少恶意应用传播、提升平台安全性；反对者担忧会严重损害 Android 的开放性，并伤害独立开发者和国际市场用户权益。也有分析认为该政策将进一步加剧 Google Play 与国内应用市场竞争格局的分化。

### [苹果发布 iOS 26.5 正式版，修复 52 个漏洞](https://zhuanlan.zhihu.com/p/2037503963512189854)
**摘要**：iOS 26.5 安全更新共修复 52 个安全漏洞，涵盖内核、WebKit、蓝牙、CoreAudio 等多个系统组件，其中多个漏洞存在被活跃利用的风险。苹果建议所有用户尽快升级，企业用户和安全敏感场景尤为重要。这是近期 iOS 更新中安全修复数量最多的一次，凸显了苹果对系统安全问题的持续重视。

### [iOS 26.5 正式版发布，你更新体验了吗？新版本有哪些亮点？](https://www.zhihu.com/question/2037481386324358180)

### [iOS/iPadOS 26.4.2 正式版发布，解决通知残留问题](https://zhuanlan.zhihu.com/p/2030706411789886155)
**摘要**：在 iOS 26.5 推送前不久，苹果于 5 月初紧急发布了 iOS/iPadOS 26.4.2，主要解决了通知横幅无法正确消失的"通知残留"问题，该 bug 在 26.4.1 版本中引发较多用户投诉。此次快速响应体现了苹果在用户体验细节上的维护能力，也印证了 26.5 将以功能更新为主，延续紧凑的迭代节奏。

### [华为昇腾 950PR 正式量产！7 万定价打穿英伟达，国产 AI 芯片终于站起来了](https://zhuanlan.zhihu.com/p/2026893491125467122)

### [科技狂欢，算力躺赢？](https://zhuanlan.zhihu.com/p/2035779347521479710)
