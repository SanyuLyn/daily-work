# 技术日报 2026-05-27

## Flutter

### [Flutter 鸿蒙 2026 路线发布，加速同步官方生态，进一步优化体验](https://zhuanlan.zhihu.com/p/2016571565735682780)
**摘要**：Flutter 鸿蒙社区发布 2026 年路线规划，核心目标是将 Flutter-OH 与 Flutter 官方版本的同步周期从过去平均 7 个月以上压缩到约 4 个月以内。同时规划至少 200 个高优先级三方库的鸿蒙适配工作，并计划为开发者提供更完善的调试工具和迁移文档，进一步降低 Flutter 开发者踏入鸿蒙生态的门槛。

### [Flutter 2026 Roadmap 发布，未来计划是什么？](https://zhuanlan.zhihu.com/p/2009657387510952563)
**摘要**：Flutter 官方发布 2026 年路线图，主要目标包括：完成 Android 平台 Impeller 渲染器迁移、在 Android 10+ 中彻底移除旧版 Skia 后端、确保对 Android 17 和最新 iOS 版本的首日支持，以及推进 Material 3 Expressive 和 Liquid Glass 风格可选组件包，进一步改善开发者工具链和 DevTools 体验。

### [新年 Flutter 3.41 就放大招，历经 3 年，多窗口支持终合并](https://zhuanlan.zhihu.com/p/2005389425048307284)
**摘要**：Flutter 3.41 正式版发布，历经 3 年开发的多窗口支持功能终于合并进入主线，得益于 Ubuntu 背后的公司 Canonical 的持续贡献。此版本同步推进了 Impeller 渲染器在 Android 平台的成熟度，并修复多项框架级 bug，标志着 Flutter 在桌面和多平台支持上迈出了重要一步。

### [Android Vulkan 官宣转正并统一渲染堆栈，这对 Flutter 又有什么影响？](https://zhuanlan.zhihu.com/p/30915375962)
**摘要**：Android 官方宣布 Vulkan 成为强制渲染标准并统一渲染堆栈，对 Flutter Impeller 渲染器意义重大。Impeller 原需同时维护 Metal（iOS）和 OpenGL/Vulkan（Android）两套后端，此次 Vulkan 转正将简化 Flutter Impeller 在 Android 上的维护路径，推动 Flutter 彻底摆脱 Skia，实现更高效统一的跨平台渲染体验。

## Android

### [Android 17 有什么需要适配的？2026 Android 禁止侧载又是什么？](https://zhuanlan.zhihu.com/p/2009937286335308608)
**摘要**：Android 17 带来多项必要适配变更，包括对 16K Page Size 的强制支持、新隐私沙箱 API、Photo Picker 强制使用等。同时 Google 正式推进 2026 年 Android 侧载限制计划，开发者需完成身份验证才能发布应用，9 月起巴西、印尼、新加坡等市场将率先执行强制验证，未经认证应用将受到安装限制。

### [Android Gradle Plugin 9.0 发布，为什么这会是个史诗级大坑版本](https://zhuanlan.zhihu.com/p/1997655149036995133)
**摘要**：AGP 9.0 引入多项破坏性变更，包括移除对多个旧版 DSL API 的支持、强制使用 Kotlin 版本目录管理依赖、编译行为调整等。文章详细梳理了升级过程中可能遭遇的各类兼容性陷阱，特别是对使用第三方插件的大型项目影响显著，建议开发团队提前规划迁移策略并充分测试后再升级。

### [Android CLI 来了！终端一键建项目、控模拟器、给 Agent 喂官方规范](https://zhuanlan.zhihu.com/p/2029843582178137700)
**摘要**：Google 发布 Android CLI 预览版，开发者可在终端完成创建项目、管理模拟器、查询官方规范等关键开发工作流，并支持 Agent 模式，可将 Android 官方知识库作为上下文提供给 AI 代理，大幅提升 AI 辅助 Android 开发效率。此工具标志着 Android 开发工具链正向命令行和 AI-first 方向加速转型。

### [Swift 官方正式支持 Android，iOS 的跨平台春天要来了吗？](https://zhuanlan.zhihu.com/p/1921602473484850131)
**摘要**：Swift 语言官方宣布成立 Android 工作组，正式维护 Android 平台支持，开发者将可使用 Swift 构建 Android 应用。这意味着 iOS 开发者未来可以用同一门语言完成双平台开发。文章探讨了该举措对 Kotlin Multiplatform、Flutter 等跨平台框架的潜在影响，以及 Swift 在 Android 生态真正落地所面临的现实挑战。
