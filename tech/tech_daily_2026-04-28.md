# 技术日报 2026-04-28

## Flutter

### [Flutter Web 正式移除 HTML renderer，只支持 CanvasKit 和 SkWasm](https://zhuanlan.zhihu.com/p/11809045697)
**摘要**：Flutter 官方宣布正式废弃 Web 端的 HTML renderer，预计年底前将变更合并至 Beta 分支，届时 `--web-renderer` 参数将彻底失效，Web 应用将统一采用 CanvasKit 或 SkWasm 渲染管线。CanvasKit 基于 Skia/WebAssembly，性能更强但包体更大；SkWasm 则是面向未来的轻量方案。开发者需尽早评估现有 Web 项目的渲染兼容性，特别是依赖 HTML renderer 特性（如特定字体渲染、CSS 混合模式）的场景需提前迁移。

### [鸿蒙版 Flutter 三方库适配案例：screen_brightness](https://zhuanlan.zhihu.com/p/29600343631)
**摘要**：文章以 `screen_brightness` 插件为例，详细介绍了如何将 Flutter 第三方库适配到鸿蒙（HarmonyOS）系统。核心流程包括：在 `pubspec.yaml` 中添加 `ohos` 平台声明、实现 `ScreenBrightnessPlugin` 的鸿蒙侧 ArkTS 代码、通过 MethodChannel 与 Flutter 层通信。文章还指出鸿蒙 Flutter 生态仍处于快速建设期，官方三方库适配仓库已收录数百个常用插件，开发者贡献适配案例可获得激励。

### [Flutter 版本选择指南：3.35 稳定，3.38 已发布](https://zhuanlan.zhihu.com/p/1977423333223048923)
**摘要**：文章系统梳理了 2025 年下半年以来 Flutter 版本的稳定性变化规律，建议生产项目在新版本发布后等待约 2 个月再升级，以规避早期 Bug。当前推荐使用 3.35 稳定版，3.38 已发布但仍处于观察期。文章还总结了各版本重点变更，包括 Impeller 渲染引擎在 Android 的默认开启、Web 编译性能优化以及新增的 DevTools 诊断功能，帮助开发者做出更合理的版本决策。

### [Flutter Web 从 0 到部署上线（实践 + 踩坑）](https://zhuanlan.zhihu.com/p/1938276051617845509)
**摘要**：作者分享了将 Flutter Web 项目从本地开发到正式上线的完整实践经验。重点涵盖：使用 `flutter build web --wasm` 编译 SkWasm 产物、Nginx 配置 `SharedArrayBuffer` 所需的 COOP/COEP 响应头、CDN 缓存策略优化首屏加载速度，以及 PWA 离线缓存配置。踩坑部分详细记录了字体资源跨域、Canvas 打印兼容性和 IndexedDB 存储限制等常见问题及解决方案，对 Flutter Web 落地有较强参考价值。

**摘要**：Google 正式推出 Android CLI 工具，配套 Android Skills（技能库）和 Android Knowledge Base（知识库），旨在为 Android 开发工程构建完整的 AI Agent 能力。Android CLI 可理解项目结构、自动执行代码生成、依赖分析和 Lint 修复等任务，与 Gemini 模型深度集成。这是继 Android Studio 内置 AI 助手后，谷歌在 Android 开发工具链 AI 化方向上的重要延伸，标志着 Android 开发工具向 Agentic 范式加速演进。

### [Android 17 Beta 4 发布：四大行为变更，不改上线就崩](https://zhuanlan.zhihu.com/p/2029480668401349168)
**摘要**：Android 17 Beta 4 正式发布，这是本轮 Beta 计划的最后一个预览版，预示正式版即将到来。文章重点梳理了四项破坏性行为变更：后台进程唤醒限制进一步收紧、JobScheduler 触发频率降低、非豁免应用的通知权限再次收严、以及针对 targetSdk < 35 应用的兼容模式调整。开发者需在 targetSdk 升级到 35+ 的前提下，逐项检查上述变更并完成适配，否则应用在 Android 17 设备上可能出现崩溃或功能异常。

### [Android 17 适配要点 + 2026 Android 禁止侧载政策解读](https://zhuanlan.zhihu.com/p/2009937286335308608)

### [如何看待谷歌逐步收紧安卓应用侧载计划？](https://www.zhihu.com/question/2019018192186557023)
**摘要**：知乎问题下汇集了多方观点，技术侧认为侧载限制有助于减少恶意软件传播，安全收益明显；产品侧则担忧国内应用市场格局受冲击，以及企业内网分发场景被波及。部分答主指出，谷歌此举与苹果在 iOS 上的管控思路趋同，但 Android 开放性是其核心竞争力，过度收紧可能加速部分市场的系统碎片化。政策目前已在 Pixel 和部分品牌新机上试行，2026 年 9 月将扩大强制执行范围。

## iOS

### [苹果 iOS / iPadOS 26 正式版推送：哪些机型支持，体验如何？](https://www.zhihu.com/question/1951229305867694703)
**摘要**：iOS 26 正式版已推送，采用全新"液态玻璃"（Liquid Glass）视觉语言，系统 UI 大幅重设计，控制中心、通知栏和 App 图标风格全面更新。支持机型为 iPhone 11 及更新机型（11 系列为最后一批），相机应用重构了拍摄界面逻辑，新增独立的"游戏"和"预览"应用。用户反馈两极分化，部分用户称视觉体验焕然一新，另有用户反映动效过于激进导致眩晕，苹果后续已在辅助功能中新增动效减弱选项。

### [iOS 27 正式淘汰 iPhone 11 系列及 SE 2：四款机型无缘升级](https://zhuanlan.zhihu.com/p/2029948433969186584)
**摘要**：据泄露信息和供应链消息，iOS 27（将随 iPhone 17 系列发布）确认放弃对 iPhone 11 系列（11 / 11 Pro / 11 Pro Max）及 iPhone SE 第二代共四款机型的支持。主要原因是这四款机型搭载 A13 仿生芯片，在端侧 AI 推理性能上已无法满足 iOS 27 新特性的硬件要求。这也意味着苹果将以 AI 能力门槛作为系统支持周期的新标准，未来机型淘汰节奏或将加速。

### [苹果重新发布 iOS 26.5 Beta 系统](https://zhuanlan.zhihu.com/p/2023833853823911201)
**摘要**：苹果撤回并重新发布了 iOS 26.5 Beta 版本，修复了首发 Beta 中被开发者广泛反馈的两处严重问题：一是 Safari 在特定条件下的内存泄漏导致浏览器无响应，二是 CarPlay 连接稳定性回退。重发版本同时带来了部分新功能预览，包括增强版"来电封锁"（支持 AI 识别骚扰电话）以及 Apple Intelligence 写作助手的多语言支持扩展。开发者可通过 Apple Developer 后台下载更新。

### [苹果 iOS 26 普及率仅 16%，系统更新策略存在哪些问题？](https://www.zhihu.com/question/1993420734807159834/answer/1995157126688040629)
**摘要**：iOS 26 发布数月后普及率仅约 16%，明显低于历史同期。分析认为主要原因有三：一是 Liquid Glass 重设计引发较强争议，持观望态度的用户增多；二是初版存在较多影响日常使用的 Bug，用户口碑传播偏负面；三是苹果 AI 功能仍需美国账号或特定地区设置才可使用，中国用户升级动力不足。答主建议苹果在推送策略上增加更多用户引导，并加快 Apple Intelligence 的本土化落地。
